Secciones y lineas generales de un documento HTML5

La especificaciones HTML5 trae muchos elementos a los desarrolladores web, permite describir la estructura de un documento web con semantica estandarizada. Este documento describe estos elementos como usarlos para definir el perfil de cualquier documento.

Problemas resueltos por HTML5

La definicion de la estructura de un documento en HTML4 y su algoritmo de perfilado es muy tosco y genera numerosos problemas:

  1. HTML5 la necesidad de elementos <div> para definir secciones semanticas sin definir valores especificos para, para los atributos class, introduciendo un nuevo elemento, <section>, el elemento de seccion HTML

  2. Mezcla varios documentos es dificil: la inclusion de un sub-documeto en un documento principal.Esto se resuelve en HMTL5 con los elementos de seccionado (<article>, <section>, <nav>, <aside>) son siempre subsecciones de su seccion ancestra mas cercana.

  3. HTML5 introduce el elemento <hgroup> que oculta todos los elementos de cabecera excepto el primero de mas alto rango(por ejemplo <hgroup><h1>Justine</h1><h2>Les malheurs de la Vertu</h2></hgroup> crea el perfil 1. Justine)
